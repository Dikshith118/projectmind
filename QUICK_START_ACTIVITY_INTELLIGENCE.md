## What You Get

Transform your basic activity tracking into a full AI-powered productivity intelligence system that:
- 📊 Tracks coding behavior in real-time
- 🎯 Detects productivity patterns
- 🔗 Correlates activity with tasks
- 📈 Estimates task completion confidence
- ⚠️ Detects stalled work
- 🤖 Generates AI-powered insights using Groq

## How It Works

### 1. Activity Tracking
VS Code extension sends events every 60 seconds:
```javascript
{
  file: "Dashboard.jsx",
  event: "save",
  timestamp: "2024-01-15T10:30:00Z"
}
```

### 2. Activity Analysis
Backend analyzes:
- Edit/save frequency
- Active coding duration
- Focus sessions (25+ min)
- Idle periods (4+ hours)
- High-intensity sessions

### 3. Task Correlation
Maps file edits to tasks:
- Uses TaskMapping model
- Estimates progress (0-100%)
- Calculates confidence scores
- Detects stalled tasks

### 4. Productivity Scoring
Generates scores (0-100):
- Daily: Duration + Sessions + Activity + Frequency
- Focus: Focus time + Session count + Avg duration
- Consistency: Active days + Avg duration + Variance
- Intensity: High-intensity sessions + Frequency

### 5. AI Insights
Groq AI analyzes all data and generates:
- Performance feedback
- Task recommendations
- Focus suggestions
- Consistency tips

### 6. Real-Time Updates
Socket.IO pushes updates:
- Activity feed refreshes instantly
- Metrics update every 60s
- Task inference updates every 30s

## Scoring Guide

### Overall Score Interpretation
- **80-100**: 🟢 Excellent - Highly productive
- **60-79**: 🔵 Good - On track
- **40-59**: 🟡 Fair - Room for improvement
- **0-39**: 🔴 Low - Needs attention

### What Affects Your Score

#### Daily Score (30% weight)
- Active coding duration (4h = 100%)
- Number of sessions (3 = 100%)
- Total events (50 = 100%)
- Edit frequency (2/min = 100%)

#### Focus Score (30% weight)
- Total focus time (2h/day = 100%)
- Focus session count (2/day = 100%)
- Average session duration (60min = 100%)
- Idle time penalty (max 20%)

#### Consistency Score (25% weight)
- Active days (7/7 = 100%)
- Average duration (2h = 100%)
- Low variance = higher score

#### Intensity Score (15% weight)
- High-intensity sessions (2/day = 100%)
- Intensity time (1h/day = 100%)
- Current frequency (3 edits/min = 100%)

## Tips to Improve Your Score

### 1. Increase Daily Score
- Code for at least 4 hours per day
- Take breaks (15min gap = new session)
- Aim for 3+ coding sessions daily
- Maintain steady edit frequency

### 2. Boost Focus Score
- Schedule 2+ deep focus sessions daily
- Each session should be 25+ minutes
- Minimize distractions during focus time
- Avoid long idle periods (4+ hours)

### 3. Improve Consistency
- Code at least 5 days per week
- Maintain similar daily durations
- Avoid large gaps between coding days
- Build a regular coding routine

### 4. Enhance Intensity
- Increase edit frequency during sessions
- Work on multiple related files
- Save frequently (shows progress)
- Maintain high-intensity bursts

## Troubleshooting

### No Activity Showing
1. Check VS Code extension is running
2. Verify `projectId` and `token` in VS Code settings
3. Check backend logs for activity events
4. Ensure MongoDB is connected

### Socket.IO Not Connecting
1. Check server is running on port 4000
2. Verify `VITE_API_URL` in client `.env`
3. Check browser console for connection errors
4. Ensure CORS is configured correctly

### AI Insights Not Generating
1. Verify `GROQ_API_KEY` in server `.env`
2. Check Groq API quota (14,400 requests/day)
3. Review server logs for errors
4. Fallback insights should still work

### Scores Showing 0
1. Need at least some activity data
2. Wait for VS Code extension to send events
3. Check ActivityLog collection in MongoDB
4. Refresh the analytics page
